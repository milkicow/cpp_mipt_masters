# Анализ функции `test`

## Исходный код

```cpp
void test(Person const& person) {
    std::cout << "test : " << person.name() << '\n';

    if (person.grade() == 10 || person.salary() > 1'000'000) {
        save(Status::success, person.id());
    } else {
        save(Status::failure, person.id());
    }
}
```

## Ветвление пути выполнения

| Элемент | Описание |
|---|---|
| `if / else` | Главный источник ветвления: выбирается одна из двух ветвей |
| `person.grade() == 10` | Результат `bool` участвует в условии |
| `\|\|` (ленивое вычисление) | `person.salary()` **не вызывается**, если `grade() == 10` уже истинно |
| `person.salary() > 1'000'000` | Результат `bool` участвует в условии (только если первое условие ложно) |

## Возможные исключения

| Элемент | Причина |
|---|---|
| `person.name()` | Метод пользовательского класса может бросить исключение |
| `std::cout << ...` | Бросает `std::ios_base::failure` при установленных флагах `exceptions()` |
| `person.grade()` | Метод пользовательского класса может бросить исключение |
| `person.salary()` | Метод пользовательского класса может бросить исключение |
| `person.id()` | Метод пользовательского класса может бросить исключение |
| `save(...)` | Свободная функция может бросить исключение (например, при ошибке I/O) |

> `Status::success` и `Status::failure` — обращение к `enum class`, исключений не бросает.
